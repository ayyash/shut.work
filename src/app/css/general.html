General

.page (responsive page container)

This class takes the maximum adequate horiztonal spaced according to browser width. .contentwrapper adds padding to the box. Without this block, the text spans the whole available width. The following are the screen stubs supported by default and the page width assigned to them (Mobile first):

  • Min
    auto
  • 480px
    auto
  • 640px
    auto
  • 768px
    620px
  • 1024px
    940px
  • 1280px
    1180px
  • 1600px
    1180px (Expanding content beyond this amount makes it hard to read)
Example layout

The idea is very simple, you only need one class to take care of responsive width, which is the page class. Following is a simple example of the most usably layout.

<div class="page">
	<h2>ShutLayout</h2>
	<p>Content is contained inside a <strong>page</strong> class element</p>
</div>
<div style="background-color:#f3f3f3">
  <div class="page contentwrapper">
    <h3>Page content</h3>
    <p>This is content in another <strong>page</strong> class element, surrounded by a different background.</p>
  </div>
</div>
<header>
  <div class="page logowrapper">
    <h1 class="sitelogo"><a href="#" title="SiteTitle">SHUT</a></h1>
  </div>
  <div class="page navwrapper">
    <nav class="main">
      <ul>
        <li><a>some element</a></li>
        <li><a>header element</a></li>
      </ul>
    </nav>
    <nav class="tools">
      <ul>
        <li><a>tool element</a></li>
        <li><a>tool element</a></li>
      </ul>
    </nav>
  </div>
</header>
<footer>
  <div class="page footerwrapper">
    <ul>
      <li>copyrights</li>
      <li><a>Footer link</a></li>
    </ul>
  </div>
</footer>
